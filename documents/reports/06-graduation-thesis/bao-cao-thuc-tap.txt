1. Giới thiệu về nội dung thực tập

Trong bối cảnh chuyển đổi số giáo dục đang diễn ra mạnh mẽ tại Việt Nam và
trên thế giới, nhu cầu về các nền tảng học trực tuyến linh hoạt, có khả năng
mở rộng và tùy chỉnh cao ngày càng tăng. Các giải pháp hiện tại như Google
Classroom, Moodle, hay các LMS truyền thống thường gặp hạn chế về khả năng
tùy biến theo đặc thù từng tổ chức giáo dục, đặc biệt là với các giảng viên
độc lập hoặc tổ chức giáo dục nhỏ và vừa.

Đợt thực tập này tập trung vào việc nghiên cứu, thiết kế và phát triển
KiteClass Platform - một nền tảng quản lý lớp học trực tuyến theo kiến trúc
Microservices, cho phép khách hàng nhanh chóng khởi tạo và vận hành hệ thống
lớp học trực tuyến riêng với giao diện và thương hiệu cá nhân hóa, được hỗ
trợ bởi trí tuệ nhân tạo.

Đối tượng hướng đến của hệ thống bao gồm:
- Các giáo viên dạy thêm cấp THCS, THPT

2.1. Lựa chọn kiến trúc

2.1.1. Phân tích các phương án kiến trúc

a) Monolithic Architecture:
- Toàn bộ ứng dụng trong một codebase
- Dễ phát triển ban đầu, khó scale và maintain khi lớn

b) Microservices Architecture:
- Chia nhỏ thành các services độc lập
- Dễ scale, maintain, nhưng phức tạp hơn trong triển khai

c) Modular Monolith:
- Monolith với các module tách biệt rõ ràng
- Cân bằng giữa đơn giản và khả năng tách service sau này

Bảng 2.1. So sánh kiến trúc Monolith vs Microservices

| Tiêu chí            | Monolith    | Microservices | Modular Monolith |
|---------------------|-------------|---------------|------------------|
| Độ phức tạp ban đầu | Thấp        | Cao           | Trung bình       |
| Khả năng scale      | Thấp        | Cao           | Trung bình       |
| Chi phí vận hành    | Thấp        | Cao           | Trung bình       |
| Độc lập deploy      | Không       | Có            | Không            |
| Phù hợp team nhỏ    | Có          | Không         | Có               |

2.1.2. Quyết định kiến trúc cho KiteClass Platform

Sau khi phân tích, chúng tôi quyết định sử dụng kiến trúc hybrid:

KiteHub Platform: Modular Monolith
- Lý do: Là hệ thống quản lý trung tâm, không cần scale từng phần riêng biệt
- Bao gồm các module: Sale, Message, AI Agent, Maintaining

KiteClass Instances: Microservices (3-5 services)
- Lý do: Mỗi instance cần scale độc lập theo nhu cầu khách hàng
- Các services có thể bật/tắt theo gói dịch vụ


2.2. Thiết kế tổng quan hệ thống

2.2.1. Kiến trúc tổng quan

Hệ thống KiteClass Platform được thiết kế với 2 thành phần chính:

A. KiteHub Platform (Modular Monolith):
- Message Module: Chat realtime với khách hàng
- Sale Module: Quản lý đơn hàng, thanh toán
- AI Agent Module: Tạo branding assets tự động
- Maintaining Module: Quản lý và provision instances

B. KiteClass Instance (Microservices):
- User + Gateway Service: Xác thực, phân quyền, routing
- Core Service: Quản lý lớp học, học viên, bài giảng
- Engagement Service (optional): Gamification, điểm thưởng
- Frontend: Next.js SSR application

Bảng 2.2. Danh sách các services trong hệ thống

| Service          | Công nghệ       | Chức năng chính                    |
|------------------|-----------------|-----------------------------------|
| KiteHub          | Java Spring     | Quản lý trung tâm, AI Agent       |
| User+Gateway     | Java Spring     | Auth, routing, user management    |
| Core Service     | Java Spring     | Class, course, content management |
| Engagement       | Java Spring     | Gamification, achievements        |
| Frontend         | Next.js 14      | SSR web application               |

2.2.2. Quy trình hoạt động

Quy trình mở node KiteClass mới:

Bước 1: Khách hàng đăng ký
- Truy cập KiteHub, chọn gói dịch vụ
- Upload ảnh cá nhân/logo tổ chức
- Thanh toán qua VietQR

Bước 2: AI Agent tạo branding (~30 giây)
- Remove.bg: Xóa nền ảnh
- Color extraction: Trích xuất bảng màu
- GPT-4: Tạo slogan, marketing copy
- Stable Diffusion XL: Tạo logo, banner
- Chi phí AI: ~$0.19/instance

Bước 3: Provision infrastructure (~3-5 phút)
- Tạo database riêng (PostgreSQL)
- Tạo storage bucket (MinIO/S3)
- Deploy services lên Kubernetes

Bước 4: Bàn giao
- Gửi email thông báo hoàn thành
- Cung cấp URL: https://[tên].kiteclass.com
- Tài khoản admin mặc định


2.3. Thiết kế cơ sở dữ liệu

2.3.1. Chiến lược Database

Áp dụng Database-per-tenant:
- Mỗi KiteClass instance có database riêng
- Đảm bảo isolation và bảo mật dữ liệu
- Dễ dàng backup/restore từng tenant

2.3.2. Schema chính của KiteClass Instance

a) Module User Management:
- users: Thông tin người dùng
- roles: Vai trò (Admin, Teacher, Student, Parent)
- permissions: Quyền hạn chi tiết
- user_roles: Mapping user - role

b) Module Class Management:
- classes: Thông tin lớp học
- class_students: Học viên trong lớp
- schedules: Lịch học
- attendances: Điểm danh

c) Module Content Management:
- courses: Khóa học
- lessons: Bài học
- materials: Tài liệu, video
- assignments: Bài tập
- submissions: Bài nộp của học viên

d) Module Assessment:
- quizzes: Bài kiểm tra
- questions: Câu hỏi
- answers: Đáp án
- quiz_attempts: Lượt làm bài
- grades: Điểm số

e) Module Engagement (optional):
- achievements: Thành tích
- badges: Huy hiệu
- points: Điểm thưởng
- leaderboards: Bảng xếp hạng


2.4. Thiết kế API

2.4.1. API Design Principles

- RESTful API với JSON response
- Versioning: /api/v1/...
- Authentication: JWT Bearer token
- Rate limiting: 100 requests/minute
- Pagination: cursor-based

2.4.2. Các API endpoints chính

a) Authentication:
POST   /api/v1/auth/register
POST   /api/v1/auth/login
POST   /api/v1/auth/refresh
POST   /api/v1/auth/logout

b) Users:
GET    /api/v1/users
GET    /api/v1/users/{id}
PUT    /api/v1/users/{id}
DELETE /api/v1/users/{id}

c) Classes:
GET    /api/v1/classes
POST   /api/v1/classes
GET    /api/v1/classes/{id}
PUT    /api/v1/classes/{id}
DELETE /api/v1/classes/{id}
GET    /api/v1/classes/{id}/students
POST   /api/v1/classes/{id}/attendance

d) Courses & Lessons:
GET    /api/v1/courses
POST   /api/v1/courses
GET    /api/v1/courses/{id}/lessons
POST   /api/v1/courses/{id}/lessons
GET    /api/v1/lessons/{id}/materials

e) Assignments:
GET    /api/v1/assignments
POST   /api/v1/assignments
GET    /api/v1/assignments/{id}/submissions
POST   /api/v1/assignments/{id}/submit


2.5. Tích hợp AI Agent

2.5.1. Các AI Services sử dụng

a) OpenAI GPT-4:
- Chức năng: Tạo slogan, marketing copy, mô tả khóa học
- Chi phí: ~$0.03/1K tokens input, ~$0.06/1K tokens output
- Ví dụ prompt:
  "Tạo 3 slogan ngắn gọn cho trung tâm tiếng Anh có tên [tên],
   phong cách chuyên nghiệp, hướng đến đối tượng [đối tượng]"

b) Stability AI - Stable Diffusion XL:
- Chức năng: Tạo logo, banner, thumbnail
- Chi phí: ~$0.002/image
- Tạo 3 phiên bản logo + 5 banner = ~$0.016

c) Remove.bg:
- Chức năng: Xóa nền ảnh upload
- Chi phí: ~$0.09/image (HD)
- Tạo ảnh transparent cho logo

Bảng 2.3. Chi phí AI Agent cho mỗi instance

| Service              | Số lượng      | Chi phí      |
|----------------------|---------------|--------------|
| GPT-4 (text)         | ~2000 tokens  | ~$0.06       |
| SDXL (images)        | 8 images      | ~$0.016      |
| Remove.bg            | 1 image       | ~$0.09       |
| Color extraction     | Local         | $0           |
| TỔNG                 |               | ~$0.19       |

2.5.2. Quy trình AI Agent

Input: Ảnh upload + Thông tin khách hàng (tên, đối tượng, phong cách)

Processing:
1. Remove background → Ảnh transparent
2. Extract colors → Primary, secondary, accent colors
3. Generate text → Slogan, description, marketing copy
4. Generate images → Logo (3 versions), Banner (5 sizes)

Output: Branding package ready to apply
